# 你的 Obsidian 工作流

目标不是做第二大脑玄学，而是让信息持续变成资产，而且能按账户、项目、站点、会议、问题持续追踪。

## 你的四条主线

1. 日常技术积累：进入 `03 Knowledge/Tech`
2. 行业知识积累：进入 `03 Knowledge/Industry`
3. 客户积累：进入 `04 Clients`
4. 项目积累：进入 `05 Projects`

## 现在这套结构怎么理解

1. `Account`
   一个公司或客户主体，比如 `GreyOrange`
2. `Contact`
   这个账户下的具体人
3. `Project / Site`
   一个明确机会、部署站点或交付对象
4. `Meeting`
   一次沟通事件
5. `Issue`
   一个支持问题、质量问题或现场问题

## 每天怎么用

1. 早上打开 `00 Dashboard/Start Here`
2. 建当天日记，写 3 个重点
3. 任何临时信息先扔进 `01 Inbox/收件箱`
4. 遇到可复用内容，沉淀成正式笔记，不要只留在日记里
5. 下班前清空收件箱，补上项目下一步
6. 用 Git 做一次备份

## 记笔记原则

- 一条笔记只讲一件事
- 能写结论就别只存链接
- 账户、联系人、项目、会议、issue 都要互相链接
- 重要信息优先加 YAML 字段，不要全靠正文搜索

## 什么时候建什么笔记

- 技术方案、踩坑、代码思路：`03 Knowledge/Tech`
- 市场观察、竞品、政策、趋势：`03 Knowledge/Industry`
- 公司、联系人、需求、推进情况：`04 Clients`
- 一个明确交付目标或部署站点：`05 Projects`
- 一次沟通或访谈：`06 Meetings`
- 一个支持问题、现场问题或缺陷：`08 Issues`

## 第一周别做的事

- 别一开始搞几十个标签体系
- 别一开始搞十几种模板
- 别试图把所有旧资料一次性迁进去
- 别把 AI 当主流程，先把你的结构搭稳
